# Contributing

Thank you for your interest in contributing to the **Sustainable-LLM-Inference-Survey** repository.

This repository is the companion resource for our systematic literature review:

> **Towards Sustainable Large Language Model Inference: A Systematic Literature Review**

Our goal is to maintain a comprehensive, accurate, and up-to-date collection of research on sustainable and energy-efficient Large Language Model (LLM) inference.

---

# How You Can Contribute

We welcome contributions including:

- Adding newly published papers
- Updating arXiv papers with published versions
- Fixing broken paper or code links
- Adding official GitHub repositories
- Correcting paper metadata (title, venue, DOI, year)
- Improving taxonomy organization
- Correcting categorization of optimization techniques
- Reporting issues or suggesting improvements

---

# Repository Scope

Contributions should be related to one or more of the following topics:

- Input-Level Optimization
- Model-Level Optimization
- Deployment-Level Optimization
- Hardware-Level Optimization
- Cross-Level Optimization
- Energy Measurement
- Carbon Measurement
- Energy-Aware LLM Serving
- Green AI
- Sustainable AI
- LLM Inference Optimization

---

# Adding a New Paper

Please add papers under the appropriate optimization category.

Use the following format:

```markdown
- **Paper Title**
  [[Paper](DOI or paper link)]
  [[Code](GitHub link)] *(optional)*
  Technique: Brief description of the proposed optimization.
```

Example

```markdown
- **AWQ: Activation-Aware Weight Quantization for LLM Compression and Acceleration**
  [[Paper](https://doi.org/xxxxx)]
  [[Code](https://github.com/xxxxx)]
  Technique: Activation-aware low-bit weight quantization for efficient LLM inference.
```

---

# Contribution Guidelines

Before submitting a Pull Request, please ensure that:

- The paper is not already included.
- The paper belongs to the scope of sustainable LLM inference.
- Links are valid and publicly accessible.
- Paper titles match the official publication.
- DOI links are preferred over PDF links whenever available.
- Categories follow the repository taxonomy.

---

# Reporting Issues

If you find:

- Incorrect information
- Broken links
- Duplicate entries
- Wrong categorization

please open a GitHub Issue describing the problem.

---

# Code of Conduct

Please be respectful and constructive when contributing.

We welcome contributions from researchers, students, and practitioners worldwide.

---

# Citation

If you use this repository in your research, please cite our paper using the citation provided in the `CITATION.cff` file.
